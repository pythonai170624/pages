ד# 📊 פרויקט גמר: שרת FastAPI לאיסוף דאטה, אימון מודל ותחזית

## 🎯 מטרת הפרויקט
# בניית שרת FastAPI שמאפשר:
# - קבלת שורות דאטה חדשות
# - ניהול הדאטה (הוספה, מחיקה, הצגה, אתחול)
# - אימון מודל רגרסיה ליניארית על בסיס הדאטה
# - תחזית (prediction) על בסיס מודל מאומן

# 🧱 מבנה הארכיטקטורה
# - נשמר כ־CSV זמני ("data.csv")
# - כולל 3 פיצ'רים: feature1, feature2, feature3
# - ועוד עמודת מטרה target

# 🔌 Endpoints ב־API:

# ➕ POST /data
# - מוסיף שורת דאטה חדשה לקובץ
# - קלט JSON:
# {
#   "feature1": 1.2,
#   "feature2": 3.4,
#   "feature3": 5.6,
#   "target": 42.0
# }

# ➖ DELETE /data/{index}
# - מוחק שורה לפי מספר שורה בקובץ

# 📄 GET /data
# - מחזיר את כל הדאטה הנוכחי

# 🔄 POST /reset
# - מאתחל את הקובץ (מוחק את כל השורות)
# - יוצר קובץ חדש עם כותרות בלבד

# ⚙️ POST /train
# - מאמן מודל רגרסיה ליניארית (LinearRegression) על בסיס הקובץ
# - שומר את המודל כ־"model.pkl"

# 🎯 POST /predict
# - מקבל שלושת הפיצ’רים בלבד (בלי target)
# - מחזיר תחזית מהמודל האחרון שאומן
# - קלט JSON:
# {
#   "feature1": 2.5,
#   "feature2": 7.0,
#   "feature3": 1.2
# }
# - פלט:
# {
#   "prediction": 38.7
# }

# 🧰 טכנולוגיות בשימוש
# - Python 3.10+
# - FastAPI
# - Pydantic
# - Scikit-learn
# - Pandas
# - Joblib (לשמירת המודל)

# ✅ שלבי פיתוח

# 2. ניהול הקובץ "data.csv"
# 3. מימוש endpoints אחד־אחד
# 4. אימון מודל עם LinearRegression
# 5. שמירה וטעינה של המודל
# 6. תחזית לפי קלט מהמשתמש

# 💡 הרחבות אפשריות בעתיד
